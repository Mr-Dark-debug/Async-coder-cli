# async-coder 0.1.3 — Sage and Local Models

Version 0.1.3 adds a second-opinion workflow for coding agents, makes provider setup more reliable, and brings locally installed Ollama models into the normal model picker.

## Highlights

### Sage consultations

- Ask a separately configured model for an independent opinion with `/consult`.
- Let the working agent consult Sage automatically when it is uncertain, comparing approaches, or checking a risky decision.
- Keep the consultation focused by sending the question together with relevant task context, then return the advice to the working agent.
- Select a default Sage provider, model, and reasoning level with `/sage-model`.

### Guided provider and model setup

- Reuse each provider's existing credential/setup flow when Sage needs a provider that is not configured yet.
- Return to Sage setup automatically after credentials are added.
- Discover provider models from live APIs, search the model list, choose a model, set reasoning, and save the complete Sage configuration.
- Show actionable errors when credentials, endpoints, or model discovery fail.

### Local Ollama support

- Connect **Ollama (local)** independently from Ollama Cloud.
- Discover installed models from the local Ollama server.
- Select local models from the standard `/models` workflow and use them as the primary working model or as Sage.

### Command reliability

- Dispatch local slash commands before a session exists, fixing commands such as `/background` on the initial screen.
- Complete provider, model, and reasoning selection transitions in both terminal and desktop flows.

## Install or upgrade

```powershell
npm install -g @async-coder/cli@0.1.3
async-coder --version
```

The npm installer currently includes the Windows x64 runtime. Platform runtime packages under `@async-coder/binary-*` are installation details and should not be installed directly.
